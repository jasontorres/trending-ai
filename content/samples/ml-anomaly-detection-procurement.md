Corruption in government procurement costs taxpayers billions annually. At BetterGov.ph, we built a machine learning system that analyzes procurement patterns to flag potential anomalies—from inflated prices to suspicious vendor relationships. The system processes millions of transactions, identifying outliers that warrant investigation.

In this technical deep-dive, I'll walk through the architecture of our anomaly detection pipeline: how we normalized messy government data, which ML models worked (and which didn't), and how we balanced false positives with genuine discoveries. We'll cover feature engineering for procurement data, handling class imbalance in fraud detection, and building systems that government auditors actually trust and use.
